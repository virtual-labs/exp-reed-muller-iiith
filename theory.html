Reed-Muller Codes - Polynomial View of Encoding

Monomials and Polynomials over F2\mathbb{F}_2F2​

A monomial is an algebraic expression that consists of a single product of variables. For our purposes, we are interested in monomials with mmm variables, X1,X2,…,XmX_1, X_2, \dots, X_mX1​,X2​,…,Xm​, where each variable can only be present or absent. This is because we operate in the binary field F2\mathbb{F}_2F2​, where any variable XiX_iXi​ squared is just itself (Xi2=XiX_i^2 = X_iXi2​=Xi​). Thus, a monomial can be written as: M=Xj1Xj2…Xjd(1) M = X_{j_1} X_{j_2} \dots X_{j_d} \tag{1}M=Xj1​​Xj2​​…Xjd​​(1) where {j1,j2,…,jd}\{j_1, j_2, \dots, j_d\}{j1​,j2​,…,jd​} is a subset of the variable indices {1,2,…,m}\{1, 2, \dots, m\}{1,2,…,m}. The number of variables that are multiplied together in the monomial (i.e., ddd, in the above monomial) is the degree of the monomial.

A Boolean polynomial is a sum of such monomials, with coefficients also from F2\mathbb{F}_2F2​ (meaning, they are either 0 or 1). An example of a polynomial in four variables is: f(X1,X2,X3,X4)=1+X1+X3+X1X2+X2X3X4(2) f(X_1, X_2, X_3, X_4) = 1 + X_1 + X_3 + X_1X_2 + X_2X_3X_4 \tag{2}f(X1​,X2​,X3​,X4​)=1+X1​+X3​+X1​X2​+X2​X3​X4​(2) The degree of a polynomial is the highest degree among all of its monomials. In the example above, the monomial X2X3X4X_2X_3X_4X2​X3​X4​ has degree 3, which is the highest, so the polynomial has degree 3.

The Evaluation of a Polynomial

The core operation for constructing Reed-Muller codes is the evaluation of a polynomial. Given a polynomial f(X1,…,Xm)f(X_1, \dots, X_m)f(X1​,…,Xm​) and a specific binary vector v=(v1,v2,…,vm)∈F2m\mathbf{v} = (v_1, v_2, \dots, v_m) \in \mathbb{F}_2^mv=(v1​,v2​,…,vm​)∈F2m​, we can evaluate the polynomial by substituting the components of v\mathbf{v}v for the variables XiX_iXi​. Eval_v(f)=f(v1,v2,…,vm)(3) \text{Eval}\_{\mathbf{v}}(f) = f(v_1, v_2, \dots, v_m) \tag{3}Eval_v(f)=f(v1​,v2​,…,vm​)(3) The result of this evaluation will be either 0 or 1.

The complete evaluation vector of a polynomial, denoted Eval(f)\text{Eval}(f)Eval(f), is the ordered list of its evaluations for every single possible input vector in F2m\mathbb{F}_2^mF2m​. Since there are 2m2^m2m such vectors, the evaluation vector will have a length of n=2mn=2^mn=2m. We conventionally list the evaluations by taking the input vectors v∈F2m\mathbf{v} \in \mathbb{F}_2^mv∈F2m​ in lexicographical order (i.e., treating them as binary numbers from 0 to 2m−12^m-12m−1).

Reed-Muller Codes

Reed-Muller (RM) codes are a family of linear block codes specified by two parameters: mmm and rrr, denoted as RM(r,m)RM(r, m)RM(r,m).

  • The parameter mmm determines the number of variables in our polynomials, which in turn sets the codeword length to n=2mn = 2^mn=2m.
  • The parameter rrr (where 0≤r≤m0 \le r \le m0≤r≤m) specifies the maximum allowable degree for our polynomials.

The code RM(r,m)RM(r, m)RM(r,m) is then formally defined as the set of all evaluation vectors of all possible Boolean polynomials in mmm variables having a degree of at most rrr.

The number of distinct monomials of degree up to rrr determines the dimension of the code, kkk. Specifically, the message bits we wish to encode correspond to the coefficients of these monomials. The dimension is thus k=∑j=0r(mj)k = \sum_{j=0}^{r} \binom{m}{j}k=∑j=0r​(jm​).

Encoding of RM Codes

Apart from a standard generator matrix multiplication (which shall be discussed in the another experiment), the encoding process for RM(r,m)RM(r, m)RM(r,m) maps a message to a polynomial, and then generates the codeword by evaluating that polynomial.

  1. Construct the Polynomial: Use the message bits as the coefficients for the set of allowed monomials (all those with degree ≤r\le r≤r) to form a message polynomial f(X1,…,Xm)f(X_1, \dots, X_m)f(X1​,…,Xm​).
  2. Generate the Codeword: Compute the evaluation vector Eval(f)\text{Eval}(f)Eval(f) by evaluating the polynomial for all 2m2^m2m input vectors in F2m\mathbb{F}_2^mF2m​. This evaluation vector is the final codeword c\mathbf{c}c.
